Distribution CLI for Agent Skills

One source.
Any harness.

Write a skill once. skillprism installs from Git, templates with MiniJinja, and emits a compliant SKILL.md for every harness you configure. No forked copies. No drift.

Quickstart Docs

From repo to every harness path

Three commands. One compile path. Order matters because each step feeds the next.

  1. skillprism add

    Install

    Pull a skill repo, detect its format, and render once per harness.

  2. skill.yaml + SKILL.md

    Author

    Metadata in YAML. Body as a MiniJinja template. One source tree.

  3. skillprism build

    Compile

    Write harness-native files to the paths each client already watches.

Manage remote skills like packages

Installed skills live in a local state file. Updates run a cheap git ls-remote check before pulling.

State: ~/.config/skillprism/installed.yaml

shell 
# install a skill from GitHub
$ skillprism add owner/repo

# list what is installed
$ skillprism list

# update when upstream moves
$ skillprism update

# remove a skill
$ skillprism remove my-skill

One compile, many destinations

Configure the harnesses you need in skillprism.yaml. Each gets its own paths, macros, and output layout.

harnesses:
  - claude
  - opencode
  # …built-in or custom harnesses

Compliant by construction

Every render includes the name and description frontmatter the Agent Skills specification requires. skillprism validate checks format, length, and emptiness before write.

frontmatter
name · description
validate
format · length · empty
render
MiniJinja · once per harness
output
SKILL.md · sidecars · manifests

Compile your first project

$ cargo install --path .
$ skillprism init project my-skills
$ cd my-skills && skillprism build
